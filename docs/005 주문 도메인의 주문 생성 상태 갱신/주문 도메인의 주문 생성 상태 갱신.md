# 주문 도메인의 주문 상태 갱신

- 주문 도메인에서 주문 상태는 따로 관리되어야 한다.

1. 주문 생성 시점에 주문 상태를 "생성됨"으로 설정한다.
    - 주문 생성 이벤트를 발행한다.
    - 상품 도메인은 주문 생성 이벤트를 구독하여 재고 차감을 수행한다.
2. 사용자는 주문 상세 정보 입력 페이지로 이동하여 배송 주소, 쿠폰, 결제 수단 등을 입력한다.
3. 주문 상세 정보 입력 완료 시점에 주문 상태를 "정보 입력 완료"로 변경한다.
    - 주문 정보 입력 완료 이벤트를 발행한다.
    - 결제 도메인은 주문 정보 입력 완료 이벤트를 구독하여 결제 준비를 수행한다.
    - 결제 도메인은 결제 성공 이벤트를 발행한다.
4. 주문 도메인은 결제 도메인의 결제 결과 이벤트를 구독하여 주문 상태를 "결제 완료"로 변경한다.
    - 만약 결제 실패 이벤트가 발행되면 주문 상태를 "결제 실패"로 변경한다.

이는 상태 다이어그램 및 상태 패턴으로 관리되어, 새로운 상태 추가나 상태 전이 변경이 용이하도록 해야 한다.
